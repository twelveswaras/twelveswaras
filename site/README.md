# twelveswaras.com static site

The public site (twelveswaras.com), served from `site/` by GitHub Pages. Pure static, with no
build step at deploy time; the raaga pages and open data are pre-generated from a data layer.

## What's here
- `index.html`: the landing page (about, how-it-works, FAQ, embedded recognizer).
- `raaga/`: the Raaga Explorer, a browse index grouped by melakarta (`raaga/index.html`) plus one
  page per raaga (`raaga/<slug>.html`), generated by `../tools/build_pages.py`.
- `listen/`: the ear-trainer.
- `data/raagas.json`: open reference data (CC-BY-4.0), from `../tools/build_data.py`.
- `llms.txt`: a plain-text project guide for LLMs / agents.
- Icons + share card: `favicon.svg` / `.ico` / `-16` / `-32`, `apple-touch-icon.png`,
  `icon-192.png` / `icon-512.png`, `og.png`. The PNGs are rasterized from the `.svg` sources.

## Regenerate
    python -m tools.merge_enrichment   # fold the draft reference + tell/rasa into the reference JSON
    python -m tools.build_data         # -> site/data/raagas.json + site/llms.txt
    python -m tools.build_pages        # -> site/raaga/*.html + the grouped index

    # icons / OG (only when the .svg sources change), from the repo root:
    rsvg-convert -w 1200 -h 630 site/og.svg     -o site/og.png
    rsvg-convert -w 180  -h 180 site/icon.svg   -o site/apple-touch-icon.png
    rsvg-convert -w 192  -h 192 site/icon.svg   -o site/icon-192.png
    rsvg-convert -w 512  -h 512 site/icon.svg   -o site/icon-512.png
    rsvg-convert -w 32   -h 32  site/favicon.svg -o site/favicon-32.png
    rsvg-convert -w 16   -h 16  site/favicon.svg -o site/favicon-16.png
    magick site/favicon.svg -background none -define icon:auto-resize=16,32,48 site/favicon.ico

## Deploy
Pushing changes under `site/**` to `main` triggers `.github/workflows/pages.yml`, which uploads
`site/` to GitHub Pages (custom domain via `site/CNAME`). No branch-folder config needed. To
redeploy without a code change: `gh workflow run pages.yml`. Note: never `gh run rerun` a failed
Pages job (it re-uploads the artifact and fails with "Multiple artifacts"); fire a fresh run, and
space retries a few minutes apart to avoid the transient "try again later" throttle.

## Wire the recognizer
`index.html` embeds the Space at `https://twelveswaras-twelveswaras.hf.space` (deployed from
`../space/`; see `../space/DEPLOY.md`). Update the iframe `src` and `og:url` if the Space slug
differs.
